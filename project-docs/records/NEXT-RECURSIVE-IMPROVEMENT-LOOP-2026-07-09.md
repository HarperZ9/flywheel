# Next Recursive Improvement Loop

Date: 2026-07-09

## Loop objective

Convert the current scaffolded harness into an evidence-producing loop using one shared run id.

## Loop steps

1. Orient: confirm artifact directory, store root, provider list, and endpoint availability policy.
2. Map: run context inventory and benchmark execution matrix.
3. Diagnose: inspect missing provider roles, missing dataset lanes, and endpoint gaps.
4. Convert: turn missing gates into explicit benchmark or readiness tasks.
5. Prioritize: run the smallest focused task set that exercises Codex, Flywheel, and local endpoints.
6. Design: freeze the task set and metric weights before execution.
7. Execute: run focused seed only after dry plan succeeds.
8. Evaluate: generate benchmark coverage, harness comparison, and outcome reports.
9. Catalog: update capability catalog and roadmap status with artifact paths.
10. Roadmap update: move only evidence-backed workstreams forward.
11. Recursive reflection: identify one harness improvement and one environment improvement.
12. Next loop: either expand provider matrix or harden the weakest tool.

## Exit criteria

The loop is complete only when one shared run id contains command plan, benchmark artifacts, coverage, comparison, endpoint evidence, and outcome synthesis.
