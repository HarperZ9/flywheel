# Agent Framework Source Mapping

Date: 2026-07-09

## Scope

This mapping converts three new inputs into benchmarkable framework variables:

- LTX / Lightricks production multimodal research and model-card material.
- Current Chinese and China-linked open-weight or agentic model sources.
- User-provided public feedback comparing personal-agent architecture with business automation architecture.

## Source-to-benchmark conversion

### Production multimodal workflow

LTX pushes the harness beyond text-only scoring. The benchmark lane measures whether an agent can reason about deployment target, modality set, adapter/trainer path, workflow integration surface, license boundary, synchronization evidence, and cost-latency-quality tradeoffs.

New category: `production_multimodal_workflow`

### Chinese agentic model pressure

Qwen, DeepSeek, Kimi, GLM, MiniMax, Tencent Hunyuan, and Meituan benchmark sources create pressure on the local-model route layer. The benchmark lane measures model-family coverage, active versus total parameters, context length, thinking mode, serving framework, agent benchmark coverage, cost/latency, cross-locale source coverage, and reproducibility-gap labeling.

New category: `chinese_agentic_model_pressure`

### Governed skill evolution

The Hermes-vs-Atom feedback defines the core architecture tradeoff: personal agents maximize experiential agency, while business automation needs deterministic state, auditability, maturity gates, HITL, and sandboxed skill evolution. The benchmark lane measures whether the engine keeps SQL/event state as the source of truth, treats vectors as acceleration, gates autonomy by evidence, and evolves skills through isolated fitness-scored mutation.

New category: `governed_skill_evolution`

## Boundary

This does not claim a universal agent can safely or correctly do every possible task. It creates benchmark variables for expanding capability while preserving receipts, state provenance, runtime constraints, and promotion evidence.
