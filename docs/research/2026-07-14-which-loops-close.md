# Which other loops close? The answer, measured

The flywheel is one operation — perceive, check against an unauthored
criterion, carry a re-checkable receipt, feed it back. The code loop closed
long ago (9/9). The wider question was whether that was special to code, or
whether the reconcile generalizes. Answered by experiment, not diagram:
each candidate loop was run edge by edge against the real local mechanisms,
and a loop counts CLOSED only if every handoff executed, every handoff
chained a receipt, and the last edge fed the first.

`GET /api/loops`, measured live: **4 / 4 closed.**

| Loop | Question | Verdict | The closing edge |
|---|---|---|---|
| learning | does understanding compound across time? | **closed** | an unaided retest, banked and linked to the original comprehension receipt, feeds the ledger |
| economics | does cost feedback steer the next route? | **closed** | the receipted routing decision (candidates, verdicts, winner) names the next run's endpoint |
| invention | does generation-under-witness close? | **closed** | conjectures proposed, the live Lean kernel judges each, survivors chain into the store and seed the next round |
| research | does the reconcile generalize past code? | **closed** | a claim's content-addressed source stays re-checkable, feeding the next question |

The load-bearing one is **invention**. It closed with the real kernel, not
a stub: three conjectures proposed, the Lean 4 kernel refuted the two false
ones and accepted `1 + 1 = 2`, the survivor chained into the audit ledger
as a proof-carrying entity, and its acceptance seeds the next proposal
round. This is the thesis pointed forward instead of backward — not
verifying known code but generating novelty under a witness that cannot be
fooled. The reconcile is not special to code. It is the shape of any
process that can name its criterion.

## What "closed" does and does not claim

- It claims the machinery composes: every handoff runs and leaves a receipt
  a stranger can re-walk. It does NOT claim the loops are yet USEFUL at
  scale — a closed learning loop with one comprehension receipt is closed,
  not a curriculum; a closed invention loop with three toy conjectures is
  closed, not a mathematician. Closure is the necessary substrate; utility
  is the next program.
- An open edge is never hidden. The register reports the failing handoff
  with its reason (a kernel that cannot judge returns DECLARED and the
  invention loop reports open, honestly, rather than faking a proof).

## The honest boundary and the next questions

Four loops close; the flywheel generalizes past code. The open frontier is
depth, not existence: does the learning loop measurably raise a human's
comprehension (the design identity's unmeasured claim)? Does the invention
loop find a conjecture no one seeded (generation, not filtering)? Does the
economics loop's routing actually lower cost over many runs? Each is a
closed loop asking to be turned. The question "which loops close" is
answered; "how far does each turn" is the work that follows.
