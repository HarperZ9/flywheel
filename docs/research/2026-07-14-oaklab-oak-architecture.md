## What it is (relayed, moderate confidence)

A research lab (the Sutton/Alberta lineage) on the OaK architecture:
event-driven neural networks, batch-size-one learning algorithms, and
agents that learn from lived real-world experience rather than curated
static datasets. Published work spans temporal-difference learning,
continual learning, and reinforcement learning; the stated aim is
superintelligence grown from experience.

## Read against the loop (the deeper rhyme)

This is closer to the thesis beneath Flywheel than to any one mechanism.
OaK's core commitment is that the external world, not a stored dataset,
supplies the learning signal continually. That is the same shape as the
invariant this whole platform is built on: no learned model sits in the
accept path; an external check disposes. Where OaK says the world
teaches the agent, Flywheel says an oracle the model did not author
decides acceptance. Both refuse a closed loop where the system grades
itself.

Two honest distinctions keep this from being overclaimed. First, OaK is
about a model that continually updates from experience; Flywheel's
engine deliberately keeps the model OUT of the accept path and freezes
the verifier. These are compatible but not the same claim: OaK grows the
proposer, Flywheel hardens the disposer. Second, batch-size-one
continual learning is a training-time architecture; the improvement loop
here is a build-time audit discipline. The rhyme is at the level of
"who supplies the criterion," not the mechanism.

Where it touches the roadmap: the academy/learn lane (does the platform
measurably lift a learner over time?) is the surface where an
experience-driven, continual-measurement stance would pour back. If a
retention curve is the unit, OaK's framing argues for measuring the
lift on lived subsequent work, not a one-shot post-test, which the
cycle-5 retention fix (graded, spaced, decaying) already moved toward.

No claim of ours changes on this source alone; it sharpens the WHY.
